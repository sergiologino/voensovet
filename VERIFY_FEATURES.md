## 🔍 КАК ПРОВЕРИТЬ ЧТО ФРОНТЕНД ПЕРЕСОБРАЛСЯ

### Шаг 1: Откройте DevTools

1. Нажмите `F12` (или `Ctrl+Shift+I`)
2. Перейдите на вкладку **Network**
3. Обновите страницу: `Ctrl+F5` (полная очистка кэша)

### Шаг 2: Найдите главный JavaScript файл

Ищите файл вида: `index-XXXXXXXX-1234567890000.js`

**КРИТЕРИЙ УСПЕХА**:
```
✅ ЕСЛИ ВИДИТЕ TIMESTAMP (длинное число в конце имени файла) - ФРОНТЕНД ПЕРЕСОБРАЛСЯ!
❌ ЕСЛИ НЕТ TIMESTAMP - ФРОНТЕНД НЕ ПЕРЕСОБРАЛСЯ
```

Примеры:
```
❌ СТАРОЕ:  index-DxNN5oBx.js
✅ НОВОЕ:   index-A1B2C3D4-1734264720000.js  ← Timestamp!
```

### Шаг 3: Проверьте размер файла

Новый файл должен быть **больше** старого, потому что:
- ✅ Добавлена библиотека `jspdf` (~200KB)
- ✅ Добавлен шрифт Arial в base64 (~1MB)
- ✅ Добавлены функции для работы с PDF

**Старый размер**: ~1.25 MB  
**Новый размер**: ~1.45 MB (примерно)

Каждый запрос должен выглядеть так:

## ❓ ЕСЛИ ФУНКЦИЙ НЕТ

### Вариант 1: Старый кэш браузера

**Решение**:
```
1. Ctrl+Shift+Delete
2. Выбрать "Изображения и файлы в кэше"
3. Очистить
4. Обновить страницу Ctrl+F5
```

**ИЛИ**:
```
1. Открыть режим инкогнито: Ctrl+Shift+N
2. Перейти на https://voensovet.ru
3. Войти в ЛК
4. Проверить функции
```

### Вариант 2: Фронтенд не пересобрался

**Проверка**:
1. DevTools → Network
2. Найти `index-*.js`
3. Посмотреть на имя файла

**Если имя БЕЗ timestamp (длинного числа)**:
```
❌ index-DxNN5oBx.js  ← Нет timestamp! Фронтенд не пересобрался!
```

**Решение**:
- Подождать ещё 5 минут (деплой может занять до 10 минут)
- Проверить логи Timeweb на наличие ошибок сборки

### Вариант 3: JavaScript ошибка

**Проверка**:
1. DevTools → Console
2. Посмотреть на красные ошибки

**Если есть ошибки**:
- Скопировать текст ошибки
- Отправить разработчику
