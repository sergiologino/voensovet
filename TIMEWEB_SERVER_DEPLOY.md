# Деплой сервера на Timeweb APP Platform

## Структура проекта

Если у вас два отдельных приложения (клиент и сервер), на Timeweb нужно создать **два отдельных приложения**:

1. **Frontend** (React) - использует `Dockerfile` в корне проекта
2. **Backend** (Node.js) - использует `Dockerfile` в папке `server/`

## Настройка сервера на Timeweb

### 1. Создание приложения

1. В панели Timeweb создайте новое приложение
2. Подключите тот же Git репозиторий
3. В настройках сборки укажите:
   - **Путь к Dockerfile**: `server/Dockerfile`
   - **Контекст сборки**: `server/` (или корень проекта, если указан путь)

### 2. Настройка переменных окружения

В разделе "Переменные окружения" добавьте переменные **БЕЗ префикса `VITE_`**:

```
DATABASE_URL=postgresql://user:password@host:5432/dbname
JWT_SECRET=your-secret-key-here
API_KEY=your-api-key
NODE_ENV=production
PORT=3000
```

### 3. Настройка порта

- Убедитесь, что переменная `PORT` установлена (например, `3000`)
- В настройках приложения укажите порт, на котором будет работать сервер
- Timeweb автоматически пробросит порт из переменной `PORT`

### 4. Проверка структуры

Убедитесь, что структура проекта следующая:

```
project/
├── Dockerfile              # Для фронтенда
├── server/
│   ├── Dockerfile          # Для бэкенда
│   ├── package.json
│   └── src/
│       └── index.js        # Точка входа сервера
└── ...
```

## Альтернативный вариант: один Dockerfile для обоих

Если вы хотите запускать клиент и сервер в одном контейнере, можно использовать `docker-compose.yml` или создать единый Dockerfile. Но на Timeweb APP Platform проще использовать два отдельных приложения.

## Проверка работы

После деплоя проверьте:

1. **Логи приложения** в панели Timeweb
2. **Health check endpoint** (если настроен): `http://your-server-url/health`
3. **Переменные окружения** - добавьте временный endpoint для проверки:

```javascript
// Только для отладки!
app.get('/debug/env', (req, res) => {
  res.json({
    nodeEnv: process.env.NODE_ENV,
    port: process.env.PORT,
    hasDbUrl: !!process.env.DATABASE_URL,
    hasJwtSecret: !!process.env.JWT_SECRET,
    // НЕ показывайте реальные значения секретов!
  });
});
```

## Частые проблемы

### Проблема: Сервер не запускается

**Решение:**
1. Проверьте логи в панели Timeweb
2. Убедитесь, что файл `src/index.js` существует
3. Проверьте, что в `package.json` указан правильный `main` или есть скрипт `start`
4. Убедитесь, что порт указан правильно

### Проблема: Переменные окружения не читаются

**Решение:**
1. Убедитесь, что переменные добавлены в настройках приложения (не в `.env` файле)
2. Переменные должны быть **БЕЗ префикса `VITE_`**
3. Перезапустите приложение после добавления переменных
4. Проверьте логи - возможно, сервер падает из-за отсутствия обязательных переменных

### Проблема: Порт не открыт

**Решение:**
1. Убедитесь, что переменная `PORT` установлена
2. Проверьте настройки порта в панели Timeweb
3. Убедитесь, что в `Dockerfile` указан `EXPOSE ${PORT}`

## Пример полного Dockerfile для сервера

```dockerfile
FROM node:18-alpine

WORKDIR /app

# Копируем package.json и package-lock.json
COPY package*.json ./

# Устанавливаем зависимости
RUN npm ci --only=production

# Копируем исходный код
COPY src ./src

# Переменные окружения передаются из настроек Timeweb
# Они доступны через process.env во время выполнения

# Открываем порт
ARG PORT=3000
ENV PORT=${PORT}
EXPOSE ${PORT}

# Запускаем сервер
# Вариант 1: Если есть скрипт start в package.json
CMD ["npm", "start"]

# Вариант 2: Прямой запуск
# CMD ["node", "src/index.js"]
```
