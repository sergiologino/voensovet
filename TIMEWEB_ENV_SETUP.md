# Настройка переменных окружения для Timeweb APP Platform

## Как это работает

Timeweb APP Platform автоматически передает переменные окружения из настроек приложения в Docker контейнер во время сборки и выполнения.

**Важно:** 
- Для **клиентской части (React/Vite)**: переменные должны начинаться с `VITE_` и встраиваются в код во время сборки
- Для **серверной части (Node.js)**: переменные доступны через `process.env` во время выполнения (без префикса `VITE_`)

## Шаги настройки

### 1. В панели управления Timeweb

1. Перейдите в раздел вашего приложения
2. Откройте настройки приложения
3. Найдите раздел **"Переменные окружения"** или **"Environment Variables"**
4. Добавьте необходимые переменные:

```
VITE_API_URL=https://your-api-url.com
VITE_API_BASE_URL=https://your-api-base-url.com
```

**Важно:** 
- Все переменные для Vite должны начинаться с префикса `VITE_`
- Эти переменные будут доступны во время сборки Docker образа
- Они НЕ попадут в git репозиторий

### 2. Использование в коде

В вашем React коде переменные доступны через `import.meta.env`:

```typescript
// Пример использования
const apiUrl = import.meta.env.VITE_API_URL;
const apiBaseUrl = import.meta.env.VITE_API_BASE_URL;

// Проверка наличия переменной
if (!import.meta.env.VITE_API_URL) {
  console.warn('VITE_API_URL не установлена');
}
```

### 3. Локальная разработка

Для локальной разработки создайте файл `.env` в корне проекта:

```env
VITE_API_URL=http://localhost:3000
VITE_API_BASE_URL=http://localhost:3000/api
```

**Важно:** Файл `.env` уже добавлен в `.gitignore` и не будет закоммичен в git.

### 4. Проверка переменных

После деплоя на Timeweb, переменные будут встроены в собранный JavaScript код во время сборки. Вы можете проверить их наличие в консоли браузера:

```javascript
console.log(import.meta.env);
```

## Как Dockerfile использует переменные

Dockerfile настроен так, что:
1. Принимает переменные через `ARG` (если они передаются через build args)
2. Преобразует их в `ENV` для использования в процессе сборки
3. Vite автоматически подхватывает переменные с префиксом `VITE_` из окружения и встраивает их в код

**Важно:** Timeweb APP Platform обычно передает переменные окружения напрямую в ENV контейнера во время сборки. В этом случае Vite автоматически их подхватит, даже если ARG не указан. Но явное указание ARG делает процесс более прозрачным и позволяет также использовать build args при необходимости.

## Добавление новых переменных

Если вам нужно добавить новую переменную:

1. Добавьте её в панели Timeweb в раздел "Переменные окружения"
2. Обновите `Dockerfile`, добавив новый `ARG` и `ENV`:

```dockerfile
ARG VITE_NEW_VARIABLE
ENV VITE_NEW_VARIABLE=$VITE_NEW_VARIABLE
```

3. Используйте в коде: `import.meta.env.VITE_NEW_VARIABLE`

## Настройка для серверной части (Node.js)

Если у вас есть отдельный сервер (backend), который разворачивается на Timeweb:

### 1. Переменные для сервера

В панели Timeweb добавьте переменные **БЕЗ префикса `VITE_`**:

```
DATABASE_URL=postgresql://user:password@host:5432/dbname
JWT_SECRET=your-secret-key
API_KEY=your-api-key
NODE_ENV=production
PORT=3000
```

### 2. Использование в серверном коде

В Node.js коде переменные доступны через `process.env`:

```javascript
// Пример использования
const dbUrl = process.env.DATABASE_URL;
const jwtSecret = process.env.JWT_SECRET;
const port = process.env.PORT || 3000;

// Проверка наличия переменной
if (!process.env.JWT_SECRET) {
  console.error('JWT_SECRET не установлена!');
  process.exit(1);
}
```

### 3. Dockerfile для сервера

Если у вас отдельный Dockerfile для сервера, убедитесь, что переменные окружения доступны во время выполнения:

```dockerfile
FROM node:18-alpine

WORKDIR /app

# Копируем package.json и устанавливаем зависимости
COPY package*.json ./
RUN npm ci --only=production

# Копируем код приложения
COPY . .

# Переменные окружения будут доступны из настроек Timeweb
# НЕ нужно указывать их здесь - они передаются автоматически

EXPOSE 3000

CMD ["node", "src/index.js"]
```

**Важно:** 
- Для сервера переменные доступны через `process.env` во время **выполнения**
- Timeweb автоматически передает переменные из настроек в контейнер при запуске
- Не нужно указывать `ARG` или `ENV` в Dockerfile для переменных, которые передаются через настройки Timeweb

### 4. Проверка переменных на сервере

Добавьте endpoint для проверки переменных (только для отладки, удалите в production):

```javascript
// Только для разработки/отладки
app.get('/debug/env', (req, res) => {
  // Показываем только названия переменных, не значения (для безопасности)
  const envKeys = Object.keys(process.env).filter(key => 
    !key.includes('SECRET') && !key.includes('PASSWORD') && !key.includes('KEY')
  );
  res.json({ 
    envKeys,
    nodeEnv: process.env.NODE_ENV,
    // Не показываем секретные значения!
  });
});
```

## Безопасность

- ✅ Переменные окружения НЕ попадают в git репозиторий
- ✅ Они хранятся только в настройках Timeweb
- ✅ Доступны только во время сборки и выполнения
- ⚠️ Помните: переменные с префиксом `VITE_` будут видны в собранном JavaScript коде клиента (не используйте для секретов!)
- ✅ Переменные без префикса `VITE_` доступны только на сервере и безопасны для секретов

Для секретных данных (API ключи, токены, пароли БД) используйте переменные **БЕЗ префикса `VITE_`** и обращайтесь к ним через `process.env` только в серверном коде.
