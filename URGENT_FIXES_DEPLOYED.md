### 3. ✅ **Умная логика авторизации**

**Проблема**: Всегда показывалась форма "Войти"

**Решение**:
```typescript
// Проверяем localStorage - был ли пользователь раньше
const isReturningUser = localStorage.getItem('hasVisited') === 'true';

const handleAuthClick = () => {
  if (isReturningUser) {
    setShowLogin(true);        // Показываем "Войти" для вернувшихся
  } else {
    localStorage.setItem('hasVisited', 'true');
    setShowRegister(true);     // Показываем "Регистрацию" для новых
  }
};
```

**Логика**:
- 🆕 **Первый визит** → Открывается **Регистрация**
- 🔄 **Повторный визит** → Открывается **Вход**
- 💾 Сохраняется в `localStorage.hasVisited`

### 5. ✅ **Форсированный ребилд фронтенда**

**Проблема**: Docker кэшировал старый билд

**Решение**:
```typescript
// vite.config.ts
build: {
  rollupOptions: {
    output: {
      entryFileNames: `assets/[name]-[hash]-${Date.now()}.js`,     // Timestamp!
      chunkFileNames: `assets/[name]-[hash]-${Date.now()}.js`,     // Timestamp!
      assetFileNames: `assets/[name]-[hash]-${Date.now()}.[ext]`  // Timestamp!
    }
  }
}
```

**Результат**:
- ✅ Каждый билд получает **уникальный timestamp**
- ✅ Docker **НЕ МОЖЕТ** использовать кэш
- ✅ Браузер получит **новые файлы** (новые хеши)
