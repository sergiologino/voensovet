# Важно: docker-compose.yml и развертывание на Timeweb

## Проблема

Если в `docker-compose.yml` есть сервис `api` (сервер), но файлы сервера отсутствуют в репозитории, сборка Docker образа будет падать с ошибкой.

## Решение

### Вариант 1: Сервер разворачивается отдельно (рекомендуется)

Если сервер разворачивается как **отдельное приложение** на Timeweb:

1. **Убедитесь, что сервис `api` закомментирован** в `docker-compose.yml`
2. Создайте **отдельное приложение** на Timeweb для сервера
3. В настройках серверного приложения укажите:
   - Путь к Dockerfile: `server/Dockerfile`
   - Контекст сборки: `server/`

### Вариант 2: Сервер в том же репозитории

Если сервер должен быть в том же репозитории и разворачиваться вместе с фронтендом:

1. Убедитесь, что в папке `server/` есть:
   - `package.json`
   - `src/index.js` (или другой файл точки входа)
   - Все необходимые файлы сервера

2. Раскомментируйте сервис `api` в `docker-compose.yml`

3. Убедитесь, что структура проекта следующая:
   ```
   project/
   ├── Dockerfile              # Для фронтенда
   ├── docker-compose.yml
   └── server/
       ├── Dockerfile          # Для бэкенда
       ├── package.json
       └── src/
           └── index.js
   ```

## Текущая настройка

В текущем `docker-compose.yml` сервис `api` **закомментирован**, так как файлы сервера отсутствуют в репозитории.

Если вы разворачиваете сервер отдельно на Timeweb, это правильная настройка.
