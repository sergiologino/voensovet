# Настройка портов для Timeweb APP Platform

## Проблема

Ошибка при деплое: `Bind for 0.0.0.0:80 failed: port is already allocated`

**Причина**: На Timeweb APP Platform порт 80 уже используется их прокси-сервером для маршрутизации входящего трафика.

## Решение

### Вариант 1: Использовать другой порт для фронтенда (текущий)

В `docker-compose.yml` изменен порт с `80:80` на `8080:80`:

```yaml
services:
  web:
    ports:
      - "8080:80"  # Внешний порт 8080, внутренний 80
```

Timeweb будет маршрутизировать трафик на порт 8080 вашего контейнера.

### Вариант 2: Использовать только EXPOSE (рекомендуется для Timeweb)

Если Timeweb использует свою систему маршрутизации, можно убрать `ports` и оставить только `expose`:

```yaml
services:
  web:
    expose:
      - "80"  # Открываем порт для других контейнеров
```

В Dockerfile оставить:
```dockerfile
EXPOSE 80
```

Timeweb автоматически настроит маршрутизацию на этот порт.

### Вариант 3: Указать порт в настройках Timeweb

В панели Timeweb APP Platform в настройках приложения укажите:
- Порт приложения: `8080` (или тот, который используется в `ports`)

## Текущая конфигурация

```yaml
# Frontend - порт 8080 (внутри контейнера nginx на порту 80)
web:
  ports:
    - "8080:80"

# Backend - порт 3000
api:
  ports:
    - "3000:3000"
```

## Как проверить

После деплоя:
1. Frontend доступен по URL приложения (Timeweb автоматически маршрутизирует)
2. Backend доступен на порту 3000 внутри сети Docker
3. Для доступа к API с фронтенда используйте относительные пути или настройте CORS

## Альтернативное решение: Разделить приложения

Если проблемы продолжаются, разверните фронтенд и бэкенд как **отдельные приложения** на Timeweb:

1. **Приложение 1 (Frontend)**:
   - Dockerfile: корневой `Dockerfile`
   - Порт: 80 (или тот, который Timeweb назначит автоматически)

2. **Приложение 2 (Backend)**:
   - Dockerfile: `server/Dockerfile`
   - Порт: 3000
   - Укажите URL бэкенда в переменных окружения фронтенда

Это более надежный подход для production.
